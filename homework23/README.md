1. Управління сесіями та cookies
Реалізовано форму входу, де користувач вводить ім’я та вік.
Ім’я зберігається у cookies, вік — у session.
На сторінці greeting виводиться персоналізоване повідомлення:
«Привіт, {ім’я}, вам {вік} років».
Додано logout:
очищає session
видаляє cookies
Cookies автоматично оновлюються при кожному запиті (перезаписуються з новим max_age).

2.1 Оптимізація запитів у ORM
Було реалізовано 2 варіанти:
Без оптимізації:
час: ~0.417 сек
SQL-запитів: 157
З оптимізацією (select_related + prefetch_related):
час: ~0.030 сек
SQL-запитів: 2

Висновок:
оптимізація значно зменшує кількість запитів і прискорює виконання (~в 10+ разів).

3.3 Індексація
Порівняння:
До індексів:
avg_time ≈ 0.135 сек
max_time ≈ 1.32 сек
Після індексів:
avg_time ≈ 0.023 сек
max_time ≈ 0.22 сек

Висновок:
індекси значно прискорюють фільтрацію.

3.4 MongoDB vs PostgreSQL
Результат:
MongoDB: ~0.0009 сек
PostgreSQL: ~0.0068 сек
Кількість записів:
78 книг в обох базах

Висновок:
MongoDB швидше читає готові дані.
PostgreSQL зручніший для роботи зі зв’язками та складними запитами.

Загальний висновок
ORM оптимізація зменшує навантаження на базу
кешування прибирає повторні запити
Celery дозволяє не блокувати додаток
індекси значно пришвидшують пошук
MongoDB швидше для читання простих структур
